## SECTION B — Indicator Computation (B9–B12)

### B9 — KDJ computation

```python
def _compute_kdj(candles, period=9):
    K, D = 50.0, 50.0
    for i in range(len(closes)):
        if i < period - 1: continue
        h_n = max(highs[i-period+1:i+1])
        l_n = min(lows[i-period+1:i+1])
        rsv = (closes[i]-l_n)/(h_n-l_n)*100 if h_n!=l_n else 50.0
        K = 2/3*K + 1/3*rsv
        D = 2/3*D + 1/3*K
    J = 3*K - 2*D
    return K, D, J
```

Uses 2/3–1/3 exponential smoothing (standard KDJ). Falls back to `(50, 50, 50)` if `len(candles) < period`.

- `j5m` = KDJ-J on `candles_1m[:-1]` (period=9, ~9-min effective lookback on 1m data)
- `j15m` = KDJ-J on `candles_15m[:-1]`
- `j1h` = KDJ-J on `candles_1h[:-1]`

`candles[-1]` excluded to avoid including the partial open candle.

**Fast stochastic:** separate `_compute_stochastic(candles_15m, k_period=8)` call. Used as `stoch_k_fast / stoch_d_fast` at score function call sites.

g_stoch gate in direction loop:
- SHORT: `stoch_k > 75 AND stoch_k < stoch_d`
- LONG:  `stoch_k < 25 AND stoch_k > stoch_d`

These g_* variables are observation flags; the hard stoch gate is inside `score_bounce_short/long`.

### B11 — `_depth_pcts()`

```python
bid_vol = sum(b["sz"] for b in bids)   # top-20 bid notional
ask_vol = sum(a["sz"] for a in asks)
bid_pct = round(bid_vol / (bid_vol + ask_vol) * 100, 1)
```

The `g_depth` observation variable uses `DEPTH_GATE_PCT=55` but it is not enforced as a `continue` gate.

## SECTION C — Signal Scoring (C13–C18)

### C13 — `score_bounce_long()`

Gates (all must pass, returns 0 on any fail):

| Gate | Condition |
|---|---|
| j15m | `j15m < J15M_LONG_GATE (20)` |
| stoch_gate | `j5m < 20 AND j15m < 20` |
| depth | `bid_pct >= 45` |

Base score = 4. Bonuses: +2 if `j5m < 20`; +2 if `trend == "Strong Bull"`; +2 if `adx >= 40`.
Max score = 10 (HC). Fire threshold = 4. HC threshold = 10 (requires all three bonuses simultaneously).

---

### C14 — `score_bounce_short()`

Gates (all must pass, returns 0 on any fail):

| Gate | Condition |
|---|---|
| j15m | `j15m > J15M_SHORT_GATE (80)` |
| stoch_gate | `j5m > 80 AND j15m > 80` |
| depth | `(100 - ask_pct) <= 65` |

Base score = 4. Bonuses: +2 if `j5m > 80`; +2 if `trend == "Strong Bear"`; +2 if `adx >= 40`.
Max score = 10 (HC). Fire threshold = 4.

**Note:** stoch_gate requires `j5m > 80`, which also satisfies the +2 j5m bonus. Effective minimum score for any passing SHORT is **6** (4 base + 2 j5m). Score 4 and 5 are unreachable for SHORTs.

### C15 — Leverage tier (`_leverage_tier`)

| ADX | Tier | Leverage |
|---|---|---|
| >= 40 | HIGH_PROB | 10x (LEVERAGE_HIGH) |
| >= 25 | MID | 5x (LEVERAGE_MID) |
| else | LOW | 5x (LEVERAGE_LOW) |

`LEVERAGE_MID = LEVERAGE_LOW = 5`. HC/HIGH_PROB leverage capped at `LEVERAGE_MID (5x)` for non-anchor pairs — applied in `_do_open_trade` after scoring.

Anchor pairs: BTC, ETH, SOL (and their USDT variants). All other pairs at HIGH_PROB tier are capped at 5x.

`ADX_MIN_SHORT = 0` — no minimum ADX gate for SHORTs. `ADX_MIN_LONG = 20`.

### C16 — SHORT gate chain (complete, in order)

1. `_regime_block_short` check → silent `continue` (HL checks first; MEXC checks after g_* assignment — cosmetic diff, same effect)
2. `g_j15m = j15m > 80` (observation only)
3. `g_j1h = j1h > J1H_SHORT_MIN` (observation only)
4. `g_stoch = stoch_k > 75 AND stoch_k < stoch_d` (observation only)
5. `g_depth = ask_pct >= DEPTH_GATE_PCT` (observation only)
6. **BTC macro trend SHORT gate** (no `_log_gate`): `_btc_j1h > _btc_j1h_history[-10]` → silent `continue` — invisible in diagnostics
7. **`SHORT_US_HALT`**: `_cur_sess == "US"` → logs gate, continues
8. **`SHORT_EU_J1H_HIGH`**: `_cur_sess == "EU" AND j1h >= 78` → logs gate, continues
9. **`SHORT_J1H_FLOOR`**: `j1h < 45` → logs gate, continues
10. **`J1H_RANGE_FAIL`**: `j1h <= J1H_SHORT_MIN (30) OR j1h >= J1H_SHORT_MAX (85)` → logs gate, continues
11. **`RSI_FLOOR_FAIL`**: `rsi15m <= RSI15M_SHORT_MIN (35)` → logs gate, continues
12. `score_bounce_short()` called

**Redundancy noted:** Gate 9 (`j1h < 45`) supersedes Gate 10's lower bound (`j1h <= 30`). j1h 31–44 is caught by Gate 9; j1h <= 30 is also caught by Gate 9. Gate 10 lower bound is dead code. Only Gate 10 upper bound (`j1h >= 85`) remains meaningful.

## SECTION F — Exit / Trade Management (F28–F35)

### F28 — `_exit_monitor_loop()` structure

Runs continuously every `PRICE_INTERVAL_SECONDS (8s)`. Iterates `app_state.open_trades`. Per trade:
- Skip if `current <= 0` (no price)
- If price age > 90s: attempt re-fetch via `get_all_prices()`; if that fails, continues with last-known price (does not skip)
- Updates `extreme_price` (best excursion) and `adverse_price` (worst excursion) every tick

Exit checks fire in this order (first match wins, `continue` to next trade):

| Order | Mechanism | Active condition |
|---|---|---|
| 1 | SL_PROXIMITY | price within 40% of remaining gap to SL |
| 2 | KILL | adverse_pct >= 0.6% |
| 3 | SL (exchange) | price crosses sl_price |
| 4 | TP1 partial close | price crosses tp1_price |
| 5 | PEAK_DECAY_20 | be_armed + peak>=sentinel_min + cpnl < peak×0.80 |
| 6 | PEAK_DECAY_10 | tp1_hit + cpnl < peak×0.90 |
| 7 | 3C_LOWER_LOW | profitable + age>=180s + 3 consecutive lower candle PnL closes |
| 8 | 3H_LOWER_HIGH | adverse + not be_armed + age>=180s + LONG 3 descending prices |
| 9 | 3L_HIGHER_LOW | adverse + not be_armed + age>=180s + SHORT 3 ascending prices |
| 10 | TIME_ADVERSE | age>=600s + cpnl<=0 + not be_armed + mfe_r<0.15R |
| 11 | SIGNAL_EXHAUSTION | J1H decayed/risen >=10pts from extreme + peak>=sentinel_min |
| 12 | TRAILBLAZER | tp1_hit + price crosses (trail_best ± atr×0.5) |

---

### F29 — SL_PROXIMITY

Fires when price has moved **60% of the way from entry toward SL** (remaining gap to SL is 40% of total SL distance).

Logic for SHORT: `(sl - current) / entry <= (sl - entry) / entry * 0.40`
→ fires when current >= entry + (sl - entry) × 0.60

No `be_armed` requirement. No MFE check. Applies all tiers, all pairs, both directions. Fires before KILL in the check order.

Example: SHORT entry=100, SL=110. SL_PROXIMITY fires at current=106. Trade exits at 60% of the adverse move rather than waiting for the exchange SL at 110.

**ADVERSE_CUT_USD (separate mechanism):** Per-pair dollar threshold. Fires when adverse PnL <= −threshold AND MFE < $10.

| NEAR, BTC, LTC | $50 |

| Default | $60 |

### F31 — PEAK_DECAY_20 / PEAK_DECAY_10

**`be_armed` arming:** `be_price` stored in trade dict. On each tick: if `current >= be_price` (LONG) or `current <= be_price` (SHORT) → `_sh["be_armed"] = True` (permanent, never reset to False within a trade). If `be_price` is absent from the trade dict, `_be_p = 0` → `_be_crossed = False` → `be_armed` never activates → PEAK_DECAY never fires.

**Peak tracking:** Updates `peak_pnl_usd` only when `be_armed AND cpnl > current_peak AND new_candle_ts > last_peak_candle_ts`. Peak is updated at most once per 1-minute candle.

**PEAK_DECAY_20** (pre-TP1): `cpnl < peak_pnl × 0.80` (or × 0.70 for @107). Requires `be_armed`, `peak_pnl >= sentinel_min`, new candle boundary.

**PEAK_DECAY_10** (post-TP1): `cpnl < peak_pnl × 0.90`. Same guards.

**sentinel_min** = `notional × SENTINEL_MIN_PEAK_PCT`. Example: AVAX EU, notional $10,000 → sentinel_min = $29. Trade must show ≥$29 peak PnL before PEAK_DECAY arms.

`SENTINEL_MIN_PEAK_PCT_DEFAULT = 0.0018` (applies to pairs not in the dict).

---

### F32 — Candle-pattern exits

**3C_LOWER_LOW:** Requires `cpnl > 0` (profitable), age ≥ 180s, new 1-min candle boundary. Tracks rolling 3-element list of per-candle PnL snapshots. Fires when `c3 < c2 < c1` and all three are positive. Applies both directions.

**3H_LOWER_HIGH** (LONG adverse): Requires `cpnl <= 0`, `not be_armed`, age ≥ 180s. Tracks 3 consecutive candle-boundary price snapshots. Fires when `p3 < p2 < p1` (descending prices — LONG going nowhere). Triggers session_halt counter.

**3L_HIGHER_LOW** (SHORT adverse): Same requirements. Fires when `p3 > p2 > p1` (ascending prices — SHORT going against). Triggers session_halt counter.

Both 3H/3L use current price as the candle boundary proxy, sampled once per new 1-min candle.

---

### F33 — TIME_ADVERSE_EXIT ⚠️ MFE GATE EFFECTIVELY DISABLED

Fires when: `age >= 600s` AND `cpnl <= 0` AND `not be_armed` AND `mfe_r < 0.15R`.

`mfe_r = peak_pnl_usd / dollar_risk` where `dollar_risk = trade.get("dollar_risk")`.

**`dollar_risk` is NOT stored in the trade dict at open time.** It is computed on-the-fly in the dashboard display loop and not written back into the trade dict. `trade.get("dollar_risk")` returns `None` → code falls to `_mfe_r_cur = 0.0`.

Result: `0.0 < 0.15` is always true. TIME_ADVERSE fires at exactly the 10-minute mark for every adverse un-armed trade, regardless of actual MFE. The `mfe_r < 0.15` guard is functionally bypassed.

Triggers session_halt counter.

---

### F34 — SIGNAL_EXHAUSTION

**LONG:** tracks `_se_j1h_extreme[key] = max(prev, cur_j1h)`. Fires when J1H has decayed ≥ 10.0 pts below the tracked peak AND `peak_pnl_usd >= sentinel_min`.

**SHORT:** tracks `_se_j1h_extreme[key] = min(prev, cur_j1h)`. Fires when J1H has risen ≥ 10.0 pts above the tracked trough AND `peak_pnl_usd >= sentinel_min`.

**No `cpnl > 0` guard.** If a trade briefly touched `sentinel_min` then went adverse, SIGNAL_EXHAUSTION can still fire during the adverse phase. For a SHORT going against (J1H rising = price likely rising), this exits a losing trade — not necessarily wrong, but the exit reason conflates "signal exhausted while winning" with "price continuing against us."

`_se_j1h_extreme` is updated regardless of PnL sign — trough/peak tracking runs even while adverse.

Follow-up task #2 (Fix Signal Exhaustion exit logic) targets this behaviour.

---

### F35 — Session halt counter

Counter key: `f"{symbol}_{direction}_{session}"`.

**Incremented by:** KILL, SL (exchange hit), 3H_LOWER_HIGH, 3L_HIGHER_LOW, TIME_ADVERSE_EXIT.

**NOT incremented by:** SL_PROXIMITY, PEAK_DECAY_20, PEAK_DECAY_10, 3C_LOWER_LOW, SIGNAL_EXHAUSTION, TRAILBLAZER.

SESSION HALT activates after **2 adverse exits** of the same symbol+direction+session combination. Halt stored in `_session_halted` set. Gate checked at alert processing time — halted symbol+direction logs `[GATE] SESSION HALT` and skips the alert.

**Session reset:** on each scan cycle, if current session differs from `_prev_session`, all keys for the previous session are cleared from `_session_sl_counts` and `_session_halted`. Halts do not carry across session boundaries.
